# Nandar Retro Game Website

Public source backup for the Laragon-based retro game portal built around EmulatorJS, curated category pages, premium fallbacks, cover import pipelines, and optional side apps such as Pokemon VN.

This repository is intentionally prepared as an installable source snapshot, not a full media archive. The local project contains more than 100 GB of ROM/CHD data and private backup artifacts, so those parts are excluded from GitHub on purpose.

## What is included

- Frontend portal and category UI
- EmulatorJS runtime wrappers for `fc`, `gba`, `gb`, `ps1`, and `mame`
- Shared helpers, fallback artwork, and cover pipeline scripts
- Admin page and catalog maintenance scripts
- Pokemon VN isolated web app source
- Generated local cover library under `covers/`

## What is not included

- ROM libraries in `fc/games`, `gba/games`, `gb/games`, `ps1/games`, `mame/games`
- Local-only secret overrides such as `config.local.php`
- Local card-payment credentials under `pokemon-vn/napthe/credentials.local.php`
- Private backup/report trees outside the repo, such as `C:\laragon\private\...`

## Quick start

1. Clone this repo into a web root such as `C:\laragon\www\retrogamewebsite`.
2. Copy [config.example.php](config.example.php) to `config.local.php`, or set the `NANDAR_*` environment variables described in [docs/SETUP.md](docs/SETUP.md).
3. Create the main MySQL database, then import or rebuild your catalog.
4. Place your ROMs into the correct system `games/` folders.
5. Use the scripts in [scripts/README.md](scripts/README.md) to import ROMs, rebuild cover data, restore a catalog backup, and run integrity audits.

## Main folders

- `common-file/`: shared PHP helpers, images, CSS/JS fragments
- `covers/`: local cover library generated by the Libretro-based pipelines
- `fc/`, `gba/`, `gb/`, `ps1/`, `mame/`: per-system launcher/runtime folders
- `pokemon-vn/`: isolated optional Pokemon browser game
- `scripts/`: import, backup, restore, audit, and cover tooling

## Recommended setup path

Use the main portal without ROMs first, then add your own collections system by system:

1. Main portal config
2. Catalog database
3. NES / GBA / GB-GBC ROM imports
4. PS1 CHD imports
5. MAME imports
6. Cover import pipelines

Full setup notes live in [docs/SETUP.md](docs/SETUP.md).

## Maintenance commands

Common examples:

```powershell
php scripts\backup_catalog.php --all
php scripts\audit_catalog_integrity.php
php scripts\restore_catalog.php C:\path\to\catalog_backup.json --dry-run
php scripts\import_gba_roms.php C:\path\to\gba\roms --apply
php scripts\import_ps1_chd_folder.php C:\path\to\ps1\chd --apply
php scripts\cover_libretro_dry_run.php --system=gba
```

More script details are documented in [scripts/README.md](scripts/README.md).
